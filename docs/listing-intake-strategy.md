# Стратегия добавления объявлений

Проект поддерживает несколько способов попадания объявлений в анализ. Сейчас это концептуальный слой и sample/mock данные: реальный парсинг URL, scraping и подключение внешних источников не реализованы.

## 1. Автоматический мониторинг по поисковому профилю

Поисковый профиль описывает набор фильтров: бюджет, тип помещения, этаж, год дома, район, признаки арендатора и другие параметры. Такой режим нужен, когда инвестор хочет регулярно получать новые подходящие объекты по заранее заданной стратегии.

Источник таких объектов должен быть помечен как автоматический мониторинг, а сами объявления должны хранить связь с поисковым профилем.

## 2. Мониторинг по локации

Локационный мониторинг нужен, когда инвестор следит за конкретной зоной: районом, улицей, жилым комплексом, станцией метро или радиусом вокруг точки. Этот режим полезен для сравнения объектов внутри одной торговой зоны и для наблюдения за локальной конкуренцией.

## 3. Мониторинг по источнику или платформе

Платформенный мониторинг нужен, когда важно отслеживать конкретный источник: CIAN, Avito, Domclick, Yandex Realty, партнерский feed или внутреннюю базу. Такой режим помогает контролировать качество источников и понимать, где объявление было найдено.

## 4. Ручной URL / ручная подборка

Ручное добавление нужно, когда пользователь сам нашел одно или несколько объявлений и хочет отправить их в анализ без настройки постоянного мониторинга.

Ручной intake должен поддерживать:

- добавление одного URL;
- добавление нескольких URL;
- создание ручной подборки;
- отдельное отображение ручных подборок от автоматически собранных объявлений;
- явную маркировку источника как `manual`.

Ручной URL intake полезен, когда:

- инвестор нашел объявление самостоятельно;
- инвестор хочет сравнить только выбранные объявления;
- пользователь хочет быстро проверить объект перед добавлением локации или профиля в мониторинг;
- источник еще не поддерживается автоматическим парсингом.

## Sample data model

```text
manual_intake_batch:
  id
  name
  description
  created_at
  status
  urls
  total_urls
  processed_count
  failed_count
  analyzed_count
  source: "manual"
  linked_search_profile_id optional

manual_listing_url:
  id
  url
  source_detected
  status
  error_message optional
  created_at
```

Статусы:

- `draft`;
- `queued`;
- `processing`;
- `analyzed`;
- `failed`.

Важно: текущая реализация не парсит URL и не обращается к внешним площадкам. UI показывает форму и sample-список ручных подборок как продуктовую концепцию для будущей реализации.
